# Firmware Update App

Self-hosted firmware update portal for the Tactical TenKey. Uses [ESP Web Tools](https://esphome.github.io/esp-web-tools/) for browser-based flashing over USB. Hosted at `t2.samwarr.dev` with Docker on TrueNAS via Cloudflare Tunnel.

## How It Works

The web app fetches `firmware.json` and `t2.bin` directly from the public firmware repo at runtime:

```
https://raw.githubusercontent.com/sammothxc/tactical-tenkey/main/firmware/firmware.json
https://raw.githubusercontent.com/sammothxc/tactical-tenkey/main/firmware/t2.bin
```

The Docker container only serves the static HTML page, no firmware files inside it.

## Updating Firmware

1. Build in PlatformIO
2. Commit and push the updated `firmware/t2.bin` and `firmware/firmware.json` (generated by PlatformIO
3. Done, and the web app picks up changes immediately

No container restart or rebuild needed for firmware updates.

## Deployment

### First time setup

1. Push this repo to GitHub (private). The GitHub Action builds and pushes the image to GHCR automatically.

2. On your Docker console, log in to GHCR:
   ```bash
   docker login ghcr.io -u sammothxc
   ```
   Use a GitHub Personal Access Token with `read:packages` scope as the password.

3. Create a stack in Dockge with this compose:
   ```yaml
   services:
     t2-firmware-app:
       container_name: t2-firmware-app
       image: ghcr.io/sammothxc/t2-firmware-app
       restart: unless-stopped
       ports:
         - 8080:80
   ```

4. Point your Cloudflare Tunnel to your desired port for your domain.

### Updating the web app

Push changes to `main`, then GitHub Action builds a new image, then pull on TrueNAS:

```bash
docker pull ghcr.io/sammothxc/t2-firmware-app
docker compose up -d
```

Or just hit "Pull & Restart" in Dockge.
